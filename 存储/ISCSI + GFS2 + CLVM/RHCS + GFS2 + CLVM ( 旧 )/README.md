#### GFS2
```txt
是一个基于GFS的先进的集群文件系统，GFS（Global File System）自身在VFS下以本地文件系统的形式存在
它能够同步每台主机的集群FS的metadata，能够进行文件锁管理，它必须要redhat cluster suite(RHCS)的支持
GFS2可以grow，进行容量的调整（这是在CLVM的基础上进行的）
多个服务器通过网络共享存储设备，每台机器都将网络共享的磁盘看作本地的，若某设备对文件执行写操作则后来访问此文件的机器将会读到
可根据对性能或是可扩展性，或者以经济性多个原则进行不同方案的部署

GFS 主要组件：
  集群卷管理
  锁管理（本地FS在集群环境中无法实现文件系统同步而GFS可以，其使用分布式的锁协议来保证数据在同一时间内只能被1个服务器进行修改）
  集群管理
  围栏和恢复
  集群配置管理

补充：
LUCI： 是基于web的集群配置方式，通过luci可轻松的搭建一个功能强大的集群系统
CLVM： Cluster的逻辑卷管理，是LVM扩展，它允许cluster的机器使用LVM来管理共享存储（工作在C端，可使ISCSI设备同步的组建LVM）
CMAN： 分布式集群管理器（启动后提供一个组播组地址，来保证数据的一致性）
```
#### CMAN
```txt
Cluster Manager 简称CMAN
是一个分布式集群管理工具，它运行在集群的各个节点上为RHCS提供集群管理任务。
CMAN用于管理集群成员、消息和通知。它通过监控每个节点的运行状态来了解节点成员之间的关系
当集群中某个节点出现故障时节点成员关系将发生改变，CMAN及时将这种改变通知底层进而做出相应的调整。
CMAN通过监视集群节点提供一个法定节点数（quorum）。当集群中有多于一半的节点处于活跃状态时此时符合法定节点数，集群继续可用
当只有有一半或少于一半的节点处于活跃状态是则已达到不到法定节点数，此时整个集群变得不可用。
CMAN通过监控集群中节点来确定各节点的成员关系
当集群中的成员关系发生改变，CMAN会通架构中其它组件来进行相应的调整
```
#### Rgmanager
```txt
RHCS通过rgmanager来管理集群服务，rgmanager运行在每个集群节点上，在服务器上对应的进程为clurgmgrd
RHCS 的核心组件为 cman & rgmanager 其中cman为基于openais的“集群基础架构层”，rgmanager为资源管理器。
在一个RHCS集群中，高可用包含集群服务和集群资源两个方面：

  集群服务： 
      其实就是应用服务，例如apache、mysql等
      
  集群资源： 
      有很多种，例如一个IP地址、一个运行脚本、ext3/GFS文件系统等
```
